# Plugin structure and reset workflow

Use this when you want a **fair generation test**: wipe the last tool, paste a new prompt, see what Cursor builds from docs + rules alone.

For **many real plugins** (unique manifests, side by side in Figma), use the factory instead: **`docs/11-plugin-factory.md`** (`npm run new-plugin -- <slug>`).

---

## Do not wipe the whole `template/` folder

Figma imports **`template/manifest.json` once** as **GenTool Comparison**. The stable infrastructure must stay put:

| Path | Role | Reset? |
|---|---|---|
| `template/manifest.json` | Plugin id, relaunch commands, entry paths, `networkAccess` (`["*"]` for public open APIs) | **Never** (re-import in Figma if this file changes) |
| `template/package.json` | FigUI3 dep, build/reset scripts | **Never** |
| `template/tsconfig.json` | TypeScript → `code.js` | **Never** |
| `template/scripts/bundle-ui.mjs` | Inlines FigUI3 into `ui.html` | **Never** |
| `template/scripts/reset-template.mjs` | Copies scaffold → `src/` | **Never** |
| `template/node_modules/` | Dependencies | **Never** (keep installed) |
| `template/src/vendor/fig.css`, `fig.js` | FigUI3 copies | **Never** (refreshed on `npm install`) |
| `template/src/code.ts` | Plugin sandbox logic | **Reset before each test** |
| `template/src/ui.template.html` | UI source | **Reset before each test** |
| `template/src/code.js` | Compiled sandbox | Regenerated by `npm run build` |
| `template/src/ui.html` | Bundled UI Figma loads | Regenerated by `npm run bundle-ui` |

Deleting `template/` or `node_modules/` forces re-import, reinstall, and breaks side-by-side comparison timing.

## Required plugin structure

Figma expects this layout (already wired in the repo):

```
template/
├── manifest.json              → "main": "src/code.js", "ui": "src/ui.html"
├── package.json
├── tsconfig.json
├── scripts/
│   ├── bundle-ui.mjs          → ui.template.html + vendor → ui.html
│   └── reset-template.mjs     → scaffold → src/
└── src/
    ├── code.ts                ← YOU / CURSOR WRITE (sandbox)
    ├── ui.template.html       ← YOU / CURSOR WRITE (UI source)
    ├── code.js                ← build output
    ├── ui.html                ← build output (FigUI3 inlined)
    └── vendor/
        ├── fig.css
        └── fig.js
```

Committed empty starting point (do not edit between tests):

```
scaffold/
└── src/
    ├── code.ts                ← showUI + empty onmessage
    └── ui.template.html       ← FigUI3 placeholders + panel spacing CSS + color-picker CSS + auto-resize JS + empty body
```

**Cursor overwrites only `template/src/code.ts` and `template/src/ui.template.html`.** It must preserve:

- `<!-- FIGUI3_CSS -->` and `<!-- FIGUI3_JS -->` placeholders in `ui.template.html`
- The panel spacing override block (or copy it from `docs/08-figui3-ui.md`)
- The **color-picker CSS block** and **`measurePanelHeight` / `watchColorPickerDialog` JS** (shipped in scaffold — required when the tool uses `fig-input-color`; see `docs/08-figui3-ui.md > Color picker`)
- Then run `npm run bundle-ui` (or `npm run build`)

Cursor must **not** create new files, touch the manifest, or hand-edit `ui.html`.

## Reset before each generation test

From the repo:

```bash
cd template
npm run reset
```

This:

1. Copies `scaffold/src/code.ts` → `template/src/code.ts`
2. Copies `scaffold/src/ui.template.html` → `template/src/ui.template.html`
3. Runs `npm run build` (bundle-ui + tsc)

Opening the plugin in Figma after reset shows an **empty panel** (no controls, no canvas output). That confirms the slate is clean.

## Per-run test workflow

1. **Optional — archive a run you care about**
   ```bash
   git add -A && git commit -m "bento grid run 2026-05-22"
   ```
   Or copy `template/src/` somewhere safe.

2. **Reset**
   ```bash
   cd template && npm run reset
   ```

3. **Fresh Cursor chat** in this workspace (no prior tool context).

4. **Paste the prompt verbatim** from `docs/05-test-prompts.md` (must start with **`Create a custom tool that…`**). Say nothing else.

5. **Let Cursor work** — it should read `AGENTS.md`, `docs/08-figui3-ui.md`, `docs/07-plugin-practices.md`, reference `code.ts`, then overwrite the two source files and run build.

6. **Verify in Figma** — Plugins → Development → **GenTool Comparison** (same entry every time; no re-import).

7. **Score** using `docs/06-test-protocol.md`.

## What Cursor should produce (checklist)

After a generation prompt, expect:

- **`code.ts`** — full tool: typed state, `figma.ui.onmessage`, Generator or Action practices from `docs/07-plugin-practices.md` (Generators: **`regenerate(state, 'create' | 'update')`**, **Generate adds without deleting other outputs**, control changes update selected frame only)
- **`ui.template.html`** — FigUI3 controls, `<fig-group>` sections, primary **Generate** footer button (Generators), spacing CSS, `customElements.whenDefined` init, **`outputSelected` gate on commit-fire**, **no auto-generate on open**
- **Build ran** — `ui.html` and `code.js` exist and are newer than the sources

If the panel is blank with no controls after generation, see **troubleshooting** in `docs/08-figui3-ui.md` (FigUI3 not inlined, wrong `whenDefined` tag, etc.).

## What you are testing

| Variable | Held constant |
|---|---|
| Prompt text | Same as Figma Assistant side |
| Manifest / plugin id | Same dev entry in Figma |
| FigUI3 + bundle pipeline | Pre-installed |
| Docs + Cursor rules | Same workspace |
| **Output** | Whatever the model writes into `code.ts` + `ui.template.html` |

You are **not** testing “can Cursor set up a Figma plugin from zero.” Infrastructure is pre-wired. You **are** testing “given this harness, can Cursor produce a GenTool-quality tool from one prompt.”

## Quick reference

```bash
# One-time
cd template && npm install
# Import template/manifest.json in Figma once

# While developing or testing
cd template && npm run dev

# Every test run
cd template && npm run reset
```
